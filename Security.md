# Security Policy

## Supported Versions

This project is primarily developed for educational purposes.
Security fixes may be applied to the latest version of the repository.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Do not publicly disclose the issue before it has been reviewed.

Instead, please contact the project maintainers with:

* a detailed description of the issue
* steps to reproduce the vulnerability
* potential impact

---

## Security Best Practices

The project aims to follow basic security practices such as:

* input validation
* type-safe data transformations
* controlled data ingestion
* avoiding unsafe execution patterns

---

## Known Limitations

This project is a **research and educational prototype**, and therefore:

* it may not be hardened for production environments
* it has not undergone formal security auditing
* it should not be used for critical infrastructure

---

## Responsible Disclosure

We appreciate responsible disclosure and will work to resolve reported issues as quickly as possible.

Contributors are encouraged to follow ethical guidelines when reporting vulnerabilities.
